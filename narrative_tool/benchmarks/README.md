# Benchmarks narrative_tool

Un benchmark est une référence éditoriale stable pour juger le ton, la structure et la dynamique d’une scène.

Ce que ce n’est pas :
- une scène canonique à intégrer telle quelle ;
- un draft à réécrire automatiquement ;
- un substitut aux règles QA.

Comment l’utiliser :
- comparer une nouvelle scène au benchmark avant validation ;
- vérifier le ton, le rythme SMS, la présence Player et la place du sous-texte ;
- garder une base commune pour Sandra Day 1 et les scènes voisines.

Pourquoi ne pas copier mécaniquement les exemples :
- les exemples servent à orienter, pas à figer la prose ;
- la scène finale doit rester naturelle, vivante et contextuelle ;
- le benchmark doit survivre aux variations de formulation.
