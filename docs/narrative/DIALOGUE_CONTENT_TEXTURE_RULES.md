# Règles de texture — dialogues, photos, emojis, vie quotidienne

## Statut

Document transversal à appliquer à toute réécriture runtime, en particulier J5/J6 puis J7+.

Objectif : préserver un jeu vivant, sensuel, joueur et mobile, même avec une structure narrative plus solide.

Ces règles complètent :

```text
SCENARIO_SPINE_J1_J10.md
J1_J10_REVISED_SCENARIO_PLAN.md
J5_J6_REWRITE_PLAN.md
PLAYER_RESPONSE_RULES.md
ANTI_LOOP_RULES.md
ROUTE_COMPATIBILITY_MATRIX.md
```

## Principe central

```text
La structure narrative ne doit pas rendre les conversations froides.
Les personnages doivent jouer, se chercher, se taquiner, se tester, se désirer et se répondre.
```

Une scène ne doit pas seulement faire avancer un état de route.

Elle doit aussi donner l’impression que les personnages existent dans un téléphone vivant :

```text
photos ;
émojis ;
messages courts ;
relances ;
malentendus ;
private jokes ;
réactions rapides ;
silences ;
mini-provocations ;
phrases qu’on regrette presque d’avoir envoyées.
```

## Photos quotidiennes / contenu visuel

Le jeu doit rester alimenté par du contenu visuel fréquent.

Règle générale :

```text
plusieurs contenus visuels peuvent apparaître par jour,
mais chaque photo doit avoir une fonction relationnelle ou ludique.
```

Les photos ne doivent pas être seulement des preuves.

Elles peuvent être :

```text
- prétexte de contact ;
- récompense ;
- trace douce ;
- provocation ;
- preuve faible ;
- souvenir ;
- défi ;
- jalousie ;
- détail à regarder ;
- signe de confiance ;
- image trop innocente pour l’être vraiment.
```

## Quantité de photos

La règle n’est pas : “une seule photo importante”.

La règle est :

```text
le jeu doit pouvoir proposer plusieurs photos par jour,
mais toutes ne doivent pas avoir le même poids.
```

Types de photos dans une journée :

```text
1. photo légère / quotidienne ;
2. photo relationnelle ;
3. photo à détail troublant ;
4. photo preuve ou conséquence ;
5. photo rare ou très signifiante.
```

Exemple J7 possible :

```text
Sandra : photo confidence / valorisation.
Mathilde : photo chamaillerie domestique.
Pauline : photo preuve ou défi.
Raphaëlle : pas forcément de photo, ou une image rare et calme.
Marie : photo de reconquête ou photo sociale.
```

Important : toutes les routes ne doivent pas envoyer une photo chaque jour, mais le jour doit souvent contenir assez de matière visuelle pour nourrir la galerie et le sentiment de vie.

## Fonction des photos par personnage

### Sandra

Photos comme :

```text
- reconnexion ;
- confiance ;
- valorisation ;
- photo offerte parce qu’elle se sent vue ;
- photo retenue si Player esquive ;
- photo moins neutre quand la confidence est vraie.
```

Règle : Sandra n’offre pas des photos parce qu’elle cède, mais parce qu’elle se sent assez vue pour oser jouer.

### Mathilde

Photos comme :

```text
- détail sexy dans une photo innocente ;
- chamaillerie ;
- revanche ;
- photo volée ludique avec reprise de contrôle ;
- photo récompense ;
- photo complice dans la maison.
```

Règle : la photo Mathilde doit rester liée au jeu domestique et à la présence morale de Marie.

### Pauline

Photos comme :

```text
- preuve faible ;
- contrôle ;
- défi ;
- selfie provoque ;
- image gardée ;
- image qui peut se retourner contre elle ;
- photo sociale qui révèle un regard.
```

Règle : Pauline ne collectionne pas gratuitement. Elle vérifie qui accepte d’être vu autrement.

### Raphaëlle

Photos rares.

Photos comme :

```text
- pause café ;
- réunion ;
- pot de départ ;
- marche ;
- image calme ;
- signe discret de confiance.
```

Règle : une photo Raphaëlle doit compter parce qu’elle est rare, pas parce qu’elle est provocante.

### Marie

Photos comme :

```text
- reconquête ;
- visibilité sociale ;
- jalousie ;
- couple ;
- image où elle rappelle ce que Player risque de perdre ;
- image qui peut ouvrir ou fermer une discussion.
```

Règle : Marie ne doit pas être seulement photographiée comme victime. Elle peut choisir de se rendre visible.

## Emojis

Les emojis doivent être présents en nombre suffisant pour donner un vrai rythme SMS.

Ils ne doivent pas disparaître sous prétexte de gravité narrative.

Règle :

```text
plus la relation est complice, plus les emojis peuvent devenir naturels, joueurs et codés.
```

Mais leur usage doit rester différencié par personnage.

### Sandra

Rares mais chargés :

```text
🙂 😅
```

Fonction : masquer la gêne, adoucir une phrase trop vraie.

### Mathilde

Plus vive :

```text
😅 🙃 😇 🫠 🙄
```

Fonction : mauvaise foi, gêne, provocation, récupération de contrôle.

### Pauline

Contrôle joueur :

```text
😇 🙂 🙃 😅
```

Fonction : fausse innocence, défi, menace douce, provocation.

### Raphaëlle

Sobres :

```text
🙂 😅
```

Fonction : adoucir une limite ou une présence, jamais surjouer la séduction.

### Marie

Quotidienne et émotionnelle :

```text
🙂 😅 🙄
```

Fonction : tendresse, agacement, reconquête, ironie de couple.

## Évolution de la franchise

Les dialogues doivent devenir plus directs au fur et à mesure de l’évolution des relations.

Règle :

```text
plus la route avance,
moins les personnages doivent se cacher derrière des formulations neutres.
```

Mais “plus direct” ne veut pas dire “plus brutal” ou “plus explicite sans préparation”.

Progression recommandée :

```text
1. allusion ;
2. blague ;
3. demi-aveu ;
4. phrase dangereuse ;
5. limite posée ;
6. proposition claire ;
7. conséquence assumée.
```

Exemple Sandra :

```text
J1 : “Pour que tu me reconnaisses.”
J5 : “On a l’air de deux personnes qui se sont manquées.”
J7 : “Ne me dis pas trop bien ce genre de chose.”
J9 : “Ce soir, une phrase gentille et une phrase dangereuse me feraient du bien.”
```

Exemple Mathilde :

```text
J3 : “J’ai noté que tu étais de mon côté.”
J6 : “Supprime ça. Enfin attends, envoie-la-moi d’abord.”
J9 : “Un jeu que Marie ne connaît pas, ça change vite de nom.”
```

## Vivant, jeu, recherche mutuelle

Les personnages doivent donner l’impression qu’ils se cherchent entre eux.

À préserver :

```text
- relances rapides ;
- petites provocations ;
- défis ;
- blagues privées ;
- malaise qui devient humour ;
- flatteries ;
- réactions à chaud ;
- silences qui veulent dire quelque chose ;
- screenshots / photos / détails ;
- personnages qui testent une limite puis reculent ou avancent.
```

Une scène trop propre est suspecte.

Mauvais signe :

```text
chacun exprime parfaitement sa fonction narrative.
```

Bon signe :

```text
un personnage plaisante pour ne pas avouer ;
un autre répond trop vite ;
Player envoie une phrase qu’il n’est pas sûr d’assumer ;
une photo change le ton ;
un emoji contredit légèrement la phrase.
```

## Player doit participer au jeu

Player ne doit pas seulement choisir une intention abstraite.

Il doit :

```text
- flatter ;
- provoquer ;
- esquiver ;
- dire une phrase trop douce ;
- envoyer une photo ;
- demander une photo ;
- poser une limite ;
- se trahir ;
- répondre trop vite ;
- supprimer puis regretter ;
- garder puis mentir ;
- être jaloux ;
- proposer un cadre.
```

Le jeu doit faire sentir qu’il aime être désiré et qu’il teste ses limites.

## Dialogues plus directs selon relation

Chaque route doit avoir des paliers de parole.

### Début

```text
prétextes ;
photos défendables ;
blagues ;
phrases à double sens.
```

### Milieu

```text
flatteries assumées ;
photos moins neutres ;
confidence ;
défis ;
limites nommées ;
jalousie ;
allusions claires.
```

### Avancé

```text
propositions ;
choix de cadre ;
consentement explicite ;
coût émotionnel ;
désir nommé ;
mensonge nommé ;
ouverture ou fermeture de route.
```

## Règles anti-froideur

À éviter :

```text
- scènes sans emojis ;
- dialogues uniquement analytiques ;
- personnages qui parlent comme des fiches de scénario ;
- photos rares au point que la galerie paraît vide ;
- Player absent ;
- choix sans réponse incarnée ;
- scènes qui avancent l’état de route sans moment vivant.
```

À rechercher :

```text
- au moins un détail concret par scène ;
- au moins une phrase imparfaite ou défensive ;
- au moins un signe de jeu ou de tension ;
- des photos fréquentes mais hiérarchisées ;
- emojis différenciés ;
- progression vers plus de direct ;
- personnage qui teste puis observe la réaction.
```

## Validation narrative d’une scène

Avant de valider une scène, vérifier :

```text
1. Y a-t-il un pivot ou une conséquence ?
2. Y a-t-il au moins un moment vivant, pas seulement une fonction ?
3. Player répond-il vraiment ?
4. Les emojis correspondent-ils à la voix personnage ?
5. La scène propose-t-elle ou prépare-t-elle une photo / trace / image si le jour en a besoin ?
6. La relation est-elle un peu plus directe qu’avant, si elle a progressé ?
7. Les personnages se cherchent-ils vraiment, ou récitent-ils leur arc ?
```

## Application à J5/J6 rewrite

J5/J6 doivent être réécrits avec structure, mais pas asséchés.

J5 doit contenir :

```text
- Marie plus active ;
- Pauline dangereusement compréhensive ;
- photos ou selfies avec fonction ;
- emojis qui gardent le ton SMS ;
- Player qui répond et se trahit.
```

J6 doit contenir :

```text
- un acte concret de Player ;
- une photo / trace qui devient décision ;
- un jeu ou une tension vivante ;
- une réponse directe de Player ;
- au moins une relation qui devient plus directe.
```

## Règle finale

```text
Le scénario doit donner une direction.
Mais ce sont les photos, emojis, blagues, maladresses et petites provocations qui donnent envie d’y rester.
```
