# Carte des preuves et secrets

> Note de lecture : la « photo canapé / Mathilde » est désormais legacy / quarantinée.
> Le J3 Mathilde actuel doit partir d’une trace douce non-preuve (plafond / araignée / chambre récupérée), pas d’une preuve forte.

> V0.57 fixe la hiérarchie narrative principale.
> Les preuves restent valides ici, mais leur poids doit rester compatible avec une Marie centrale et des routes secondaires différées.

## Objectif

Les photos, captures et traces ne doivent pas devenir un gimmick.

Chaque preuve doit avoir une fonction précise :

```text
- qui la possède ;
- qui l’a vue ;
- ce qu’elle prouve vraiment ;
- ce qu’elle ne prouve pas ;
- quand elle change de sens ;
- quand elle doit disparaître, être révélée ou devenir dangereuse.
```

Règle :

```text
Une preuve ne doit pas revenir si elle ne change pas de fonction.
```

## Types de preuves

### Trace douce

Une trace qui trouble mais ne prouve rien.

Exemples :

```text
photo jolie
message ambigu
story sociale
souvenir d’un moment
```

Fonction : créer une gêne, pas déclencher une confrontation.

### Preuve faible

Une trace qui peut être interprétée contre Player, mais qui reste défendable.

Exemples :

```text
photo de soirée mal cadrée
capture partielle
message vu trop tard
réaction trop rapide à une photo
```

Fonction : modifier le comportement, pas condamner.

### Preuve sociale

Une trace visible ou commentée par plusieurs personnes.

Exemples :

```text
story Nico / Marie
photo de groupe Pauline
commentaire de groupe
```

Fonction : sortir le trouble du privé.

### Preuve dangereuse

Une trace qui peut créer une conséquence nette.

À réserver pour plus tard.

Exemples :

```text
image explicitement compromettante
capture d’un mensonge
contradiction horaire
preuve impliquant Pauline elle-même
```

Fonction : imposer un coût.

## Preuves actuelles

### Photo Sandra J1

Statut : trace douce.

Possesseur : Sandra / Player selon conversation.

Ce qu’elle prouve : Sandra existe comme manque et curiosité ancienne.

Ce qu’elle ne prouve pas : relation assumée, désir consommé, trahison.

Évolution possible :

```text
J5-J6 : ne pas répéter via nouvelle photo.
J7+ : Sandra peut retirer une image, refuser d’en envoyer une, ou proposer un vrai moment sans support visuel.
```

### Photo canapé / Mathilde (legacy quarantiné)

Statut : legacy historique, à ne plus traiter comme preuve actuelle J3.

Possesseur : Player si gardée ; Mathilde sait ou soupçonne qu’elle existe.

Ce qu’elle prouvait : Player a laissé une trace d’un trouble dans l’espace de Marie.

Ce qu’elle ne prouve pas : acte consommé ou route assumée.

Problème actuel : elle ne doit plus revenir comme base du J3 actuel.

Évolution obligatoire :

```text
Après J6, ne plus demander simplement “tu l’as gardée ?”.
La trace doit être supprimée, déplacée, ou transformée en conséquence domestique.
```

Options futures :

```text
- suppression réelle ;
- Mathilde demande explicitement de ne plus jouer avec ça ;
- Marie approche sans savoir exactement ;
- Player ment sur l’existence de la photo ;
- la photo cesse d’être centrale et devient un comportement.
```

### Trace douce J3 Mathilde (plafond / araignée / chambre récupérée)

Statut : trace douce non-preuve.

Possesseur : Mathilde / Player selon la scène, sans verrou de preuve.

Ce qu’elle prouve : un trouble domestique, une gêne concrète, une chambre à gérer.

Ce qu’elle ne prouve pas : un acte, une route, une sexualisation, une preuve forte.

Évolution :

```text
J3 : garder cette trace faible et respirable.
J4+ : redéfinir les preuves à partir de ce pivot, pas à partir du canapé / photo ambiguë.
```

### Photo groupe J4

Statut : preuve sociale faible.

Possesseur : Pauline / groupe / Player selon diffusion.

Ce qu’elle prouve : Player, Marie, Nico, Mathilde et Pauline existent dans un regard collectif.

Ce qu’elle ne prouve pas : intention claire.

Évolution : utile pour J5 seulement si elle crée une dette sociale.

À éviter : zoomer encore et encore sur des détails sans conséquence.

### Story Nico / Marie J5

Statut : preuve sociale.

Possesseur : réseau social / groupe.

Ce qu’elle prouve : Marie peut être visible, désirée ou remarquée par d’autres.

Ce qu’elle ne prouve pas : tromperie de Marie ou intention hostile de Nico.

Risque : Nico devient raccourci de jalousie.

Évolution : rare, défendable, jamais antagoniste pur.

### Photo Marie après soirée J5

Statut : trace de couple.

Possesseur : Marie / Player.

Ce qu’elle prouve : Marie veut être regardée, pas seulement rassurée.

Ce qu’elle ne prouve pas : qu’elle soupçonne tout.

Problème actuel : elle nourrit trop facilement le motif “pose ton téléphone”.

Évolution : ne plus centrer Marie sur le téléphone. Utiliser la photo comme point de départ vers une action différente de Marie.

### Photo Pauline soft provocation J5

Statut : preuve faible / jeu de contrôle.

Possesseur : Pauline.

Ce qu’elle prouve : Pauline teste Player et sait cadrer son propre pouvoir.

Ce qu’elle ne prouve pas : son secret complet.

Évolution : doit mener à une faille de Pauline, sinon elle devient toute-puissante.

### Seed preuve Pauline

Statut : preuve dangereuse différée.

Possesseur exact : à définir.

Ce qu’elle pourrait prouver : Pauline n’est pas seulement celle qui expose ; elle est aussi exposée.

Ne pas révéler trop tôt.

Fonction future : rendre Pauline vulnérable, pas seulement manipulatrice.

## Règles de gestion des preuves

### Règle 1 — Pas d’accumulation gratuite

Ne pas ajouter une nouvelle photo si une ancienne peut changer de sens.

### Règle 2 — Une preuve = une conséquence

Chaque preuve utilisée doit produire au moins une conséquence :

```text
choix ;
mensonge ;
silence ;
recul ;
geste concret ;
changement de disponibilité ;
perte de contrôle.
```

### Règle 3 — Pas de preuve définitive trop tôt

Avant J10, préférer :

```text
preuves faibles ;
interprétations ;
réactions ;
contradictions légères.
```

### Règle 4 — Qui sait quoi prime sur le drame

Un personnage ne doit jamais commenter une preuve qu’il n’a pas vue ou ne peut pas soupçonner.

### Règle 5 — Une preuve doit parfois mourir

Toutes les traces ne doivent pas rester actives.

Certaines doivent être :

```text
supprimées ;
oubliées ;
déplacées ;
remplacées par une conséquence comportementale.
```

## Application à la réécriture J5/J6

J5 doit sélectionner moins de preuves et leur donner plus de poids.

J6 doit contenir moins de commentaires sur les preuves et plus d’actions de Player autour d’elles.

Exemple :

```text
Mauvais : Pauline rappelle encore qu’elle garde des photos.
Meilleur : Player demande une image précise, puis Pauline refuse et ce refus le pousse à mentir à Marie / écrire à Sandra / effacer une trace.
```

Exemple :

```text
Mauvais : Mathilde demande encore si la photo existe.
Meilleur : Mathilde voit que Player n’a pas supprimé, ou le force à choisir maintenant.
```
