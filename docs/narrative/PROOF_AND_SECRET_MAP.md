## Objectif maintenu

Les photos, captures et traces ne doivent pas devenir un gimmick.

Chaque preuve doit avoir une fonction précise :

```text
- qui la possède ;
- qui l’a vue ;
- ce qu’elle prouve vraiment ;
- ce qu’elle ne prouve pas ;
- quand elle change de sens ;
- quand elle doit disparaître, être révélée ou devenir dangereuse ;
- si elle agit comme trace narrative, moteur d’excitation, objet de tromperie, preuve sociale, ou risque adulte.
```

Règle maintenue :

```text
Une preuve ne doit pas revenir si elle ne change pas de fonction.
```

## Typologie V0.68

### Trace douce

Trace qui trouble mais ne prouve rien.

Exemples :

- photo jolie ;
- message ambigu ;
- souvenir ;
- détail de travail ;
- photo ancienne ;
- objet domestique.

Fonction : créer une gêne, une mémoire, une attraction ou une possibilité sans confrontation.

### Trace érotisable

Trace encore défendable, mais que le regard de Player peut charger sexuellement.

Exemples :

- cadrage accidentel ;
- tenue ;
- photo sauvegardée ;
- message relu ;
- réaction trop rapide ;
- story sociale ;
- détail du corps ou de la pose sans intention explicite.

Fonction : faire basculer le regard, pas encore condamner.

### Preuve faible

Trace qui peut être interprétée contre Player, mais reste défendable.

Exemples :

- capture partielle ;
- message vu trop tard ;
- réponse supprimée ;
- photo gardée ;
- horaire ambigu ;
- réaction visible.

Fonction : modifier le comportement, créer dette ou mensonge.

### Preuve sociale

Trace visible ou commentée par plusieurs personnes.

Exemples :

- photo de groupe ;
- story ;
- commentaire social ;
- regard de Nico ;
- cadrage Pauline ;
- Marie visible ailleurs.

Fonction : sortir le trouble du privé.

### Image adulte choisie

Image envoyée ou montrée avec intention adulte par un personnage majeur et consentant.

Fonction : récompense adulte, confiance, provocation, bascule de route.

Règle : doit respecter la voix du personnage et son moteur NSFW.

### Preuve dangereuse / route sombre

Trace qui peut produire une conséquence nette.

Exemples :

- capture d’un mensonge ;
- image compromettante ;
- preuve impliquant Pauline elle-même ;
- contradiction horaire ;
- photo découverte par la mauvaise personne ;
- diffusion ou menace non consentie.

Fonction : imposer un coût.

Règle : non-consented exposure / blackmail ne doit pas être une récompense propre.

## Règles NSFW pour photos et preuves

### Règle 1 — Le regard peut devenir pornographique

Une trace ordinaire peut devenir adulte si :

- le contexte est chargé ;
- Player la garde / la relit / la désire ;
- le personnage choisit d’en montrer davantage ;
- la route a atteint le bon niveau d’intensité.

### Règle 2 — Consentement et agency

Une image adulte choisie est différente d’une image volée ou diffusée.

Le jeu peut explorer routes sombres, tromperie, NTR/cuckold, partage, trio/quatuor, mais doit garder :

- agency des personnages ;
- risques ;
- conséquences ;
- distinction entre fantasme consenti et dommage narratif.

### Règle 3 — Une preuve peut être excitante et dangereuse

Une trace peut être :

```text
excitation + dette + mensonge + risque
```

Mais si elle n’a qu’une fonction décorative, la couper.

### Règle 4 — Qui sait quoi reste essentiel

Un personnage ne doit jamais commenter une preuve qu’il n’a pas vue ou ne peut pas soupçonner.

### Règle 5 — Une preuve doit parfois mourir

Toutes les traces ne doivent pas rester actives.

Certaines doivent être :

```text
supprimées ;
oubliées ;
déplacées ;
remplacées par une conséquence comportementale ;
transformées en fantasme ;
transformées en coût.
```

## Carte de connaissance

La carte de connaissance concrète doit être reconstruite par jour à partir des source packs.

Ne pas utiliser les anciennes sections J5/J6/J7 comme vérité actuelle.

Avant d’écrire un jour, définir :

```text
Qui possède la trace ?
Qui l’a vue ?
Qui croit qu’elle existe ?
Qui ignore encore ?
Qui pourrait la découvrir ?
Est-elle douce, érotisable, faible, sociale, adulte choisie, ou dangereuse ?
Quel coût si elle ressort ?
```
