# Réseau Intime — 09 Progression visuelle et photographique

## Statut du document

**Catégorie : Canon**

**Niveau : Direction visuelle, photographique et pornographique de la première saison**

Ce document définit :

- la fonction narrative des images ;
- le rythme visuel minimal ;
- la progression propre à chaque personnage ;
- les différences entre photo diégétique et représentation de scène ;
- les niveaux d’intensité visuelle ;
- les audiences ;
- les règles de conservation et de diffusion ;
- l’organisation de la galerie ;
- les contrats de production et de validation ;
- les règles applicables aux contenus érotiques, sexuels et pornographiques.

# 1. Principe directeur

> **Les images ne décorent pas la progression relationnelle. Elles la rendent visible, désirable et mémorable.**

Une image importante doit permettre au joueur de comprendre au moins un changement :

- une personne se montre autrement ;
- une audience devient plus précise ;
- un regard est reconnu ;
- une version devient privée ;
- une limite est nommée ;
- une confiance est accordée ;
- une preuve apparaît ;
- un secret devient matériel ;
- une relation atteint une nouvelle intimité ;
- une conséquence devient impossible à nier.

La progression visuelle ne correspond pas simplement à :

```text
plus de vêtements
→ moins de vêtements
→ nudité
→ sexe
```

Elle correspond à :

```text
qui montre
→ quoi
→ pourquoi
→ à qui
→ avec quelle intention
→ sous quelle règle
→ avec quel risque
→ et ce que cette image signifie ensuite
```

Une image plus exposée peut représenter une progression relationnelle faible.

Une image habillée mais explicitement choisie pour Player peut représenter une bascule majeure.

---

# 2. Promesse visuelle du jeu

`Réseau Intime` est un jeu narratif adulte visual-first.

Son rythme fondamental est :

```text
la situation crée l’attente
→ les messages orientent le regard
→ la scène produit ou promet une image
→ le choix modifie l’accès, l’audience ou la signification
→ la galerie conserve ce qui peut être conservé
→ la route détermine ce que l’image pourra devenir
```

Le joueur doit avoir envie de continuer parce qu’il veut découvrir :

- quelle version chaque femme montrera ensuite ;
- si cette version lui est réellement destinée ;
- ce qu’elle sait de l’effet produit ;
- ce qu’elle autorise ;
- ce qu’elle refuse ;
- ce que l’image peut devenir dans le réseau ;
- quelle forme sexuelle ou pornographique appartient à cette route.

La crédibilité rend la récompense visuelle plus forte.

Elle ne doit pas servir de prétexte pour maintenir le jeu dans une retenue visuelle permanente.

# 3. Vocabulaire canonique

## 3.1 Contenu visuel

Terme général couvrant :

- photographie diégétique ;
- capture ou recadrage ;
- image sociale ;
- selfie ;
- photo prise par un tiers ;
- photographie de groupe ;
- représentation visuelle d’une scène vécue ;
- souvenir visuel ;
- image sexuelle ou pornographique ;
- trace visuelle supprimée ou découverte.

## 3.2 Photo diégétique

Image qui existe réellement dans l’univers.

Elle possède :

- un appareil ou moyen de capture ;
- un auteur ;
- une origine ;
- un moment ;
- une audience ;
- une règle de sauvegarde ;
- un risque de diffusion ou de découverte.

Elle peut être :

- envoyée ;
- publiée ;
- conservée ;
- supprimée ;
- recadrée ;
- transférée ;
- découverte ;
- commentée par un tiers.

## 3.3 Image de scène

Représentation destinée au joueur d’une interaction vécue.

Elle n’existe pas nécessairement comme fichier dans le téléphone des personnages.

Exemples :

- moment intime hors téléphone ;
- baiser ;
- scène sexuelle ;
- après-coup dans une chambre ;
- regard échangé pendant un événement ;
- Player présent dans une scène qu’aucun personnage ne photographie.

Une image de scène :

- peut entrer dans une galerie de souvenirs ;
- ne peut pas être envoyée ou découverte par un autre personnage ;
- n’a pas besoin d’une justification artificielle de caméra ;
- ne doit jamais être traitée comme une photographie volée.

## 3.4 Trace visuelle

Image dont la fonction première est de prouver ou rappeler qu’un moment a existé.

Elle peut être :

- imparfaite ;
- publique ;
- indirecte ;
- partielle ;
- déplacée ;
- imprimée ;
- supprimée ;
- devenue plus importante après coup.

Une trace imparfaite n’est pas une identité de personnage.

Elle doit posséder une origine, une raison d’avoir été conservée et une relation avec la personne représentée.

## 3.5 Récompense visuelle

Image ou série d’images donnant au joueur une progression lisible et désirable.

Elle n’est pas nécessairement explicite.

Elle doit cependant offrir quelque chose de nouveau :

- nouvelle version ;
- nouvelle intention ;
- nouvelle audience ;
- nouvelle proximité ;
- nouveau niveau de risque ;
- nouvelle information corporelle ou relationnelle.

## 3.6 Set visuel

Groupe cohérent de deux à six contenus visuels appartenant au même mouvement narratif.

Un set n’est pas une duplication d’une pose.

Il possède une progression interne.

## 3.7 Famille visuelle

Identité récurrente d’une route.

Exemples :

- `marie_social_reconquest` ;
- `sandra_chosen_representation` ;
- `mathilde_domestic_intent` ;
- `pauline_double_address` ;
- `raphaelle_chosen_transformation` ;
- `nico_shared_gaze_risk`.

# 4. Les cinq axes d’une image

Une image ne peut pas être définie par un seul niveau d’intensité.

Elle doit être située sur cinq axes indépendants.

## 4.1 Fonction relationnelle

Valeurs possibles :

- contexte ;
- présence ordinaire ;
- souvenir ;
- invitation ;
- attention ;
- comparaison ;
- image destinée ;
- preuve ;
- pacte ;
- dette ;
- bascule ;
- conséquence ;
- réparation ;
- clôture.

## 4.2 Intensité visuelle

Définie par les niveaux `V0–V5`.

## 4.3 Intention

Valeurs possibles :

- accidentelle ;
- ordinaire ;
- consciente de l’attractivité ;
- destinée ;
- provocante ;
- négociée ;
- explicitement sexuelle ;
- pornographique.

## 4.4 Audience

Valeurs possibles :

- publique ;
- sociale nommée ;
- privée ;
- destinée à Player ;
- partagée avec plusieurs adultes nommés ;
- secrète ;
- découverte hors audience ;
- recontextualisée sans autorisation.

## 4.5 Permanence et risque

Valeurs possibles :

- consultable publiquement ;
- conservable ;
- temporaire ;
- conservation interdite ;
- suppression demandée ;
- supprimée ;
- transférable à une audience nommée ;
- transfert interdit ;
- découverte possible ;
- déjà compromise.

Une image peut donc être :

```text
V2 sexy
+ relationnellement ordinaire
+ publique
+ conservable
+ risque faible
```

ou :

```text
V1 élégante
+ destinée uniquement à Player
+ intention clairement reconnue
+ conservation incertaine
+ risque relationnel élevé
```

La seconde peut représenter une progression beaucoup plus forte.

# 5. Échelle d’intensité visuelle

Les niveaux sont internes à la conception.

Ils ne doivent pas être affichés au joueur comme des statistiques.

## V0 — Trace ordinaire

- présence quotidienne ;
- cadrage imparfait ;
- situation concrète ;
- corps non mis en avant comme récompense ;
- fonction principale narrative ou contextuelle.

Exemples :

- ancienne photo de déjeuner que Sandra a choisi de conserver ;
- Mathilde dans la chambre encombrée ;
- Marie pendant l’installation de La Verrière ;
- Raphaëlle avec son garment bag au travail ;
- groupe social à L’Annexe.

## V1 — Élégante

- personnage clairement attirant ;
- tenue ou silhouette lisible ;
- cadrage contrôlé ;
- corps entier ou présence valorisée ;
- image socialement ordinaire ou publique.

Le joueur doit comprendre que la personne est désirable, sans ambiguïté sur la destination de l’image.

## V2 — Sexy

- corps ou pose volontairement mis en valeur ;
- jambes, taille, dos, décolleté, vêtement ajusté ou mouvement central ;
- image encore socialement défendable ;
- intention érotique possible, mais pas nécessairement destinée à Player.

## V3 — Provocante

- la personne sait que l’image peut exciter son destinataire ;
- pose, cadrage ou vêtement choisis en conséquence ;
- version privée, crop alternatif ou détail plus intime ;
- invitation, test ou reconnaissance du désir ;
- route devenue visuellement identifiable.

## V4 — Érotique adulte

- lingerie ;
- nudité ;
- exposition corporelle directe ;
- posture clairement sexuelle ;
- contexte masturbatoire ou explicitement intime sans acte pornographique complet ;
- audience et règles de conservation centrales.

## V5 — Pornographique

- acte sexuel explicite ;
- masturbation montrée frontalement ;
- oralité ;
- pénétration ;
- activité de groupe ;
- voyeurisme négocié ;
- roleplay sexuel explicite ;
- autre représentation où l’activité sexuelle est le sujet principal.

V5 reste soumis :

- au canon du personnage ;
- à la route ;
- au consentement ;
- au cadre ;
- aux audiences ;
- aux conséquences ;
- à l’après-coup.

# 6. Contrat visuel quotidien

## 6.1 Minimum

Toute journée narrative entièrement jouable doit proposer au minimum :

```text
3 contenus visuels distincts
centrés principalement sur une ou plusieurs femmes adultes principales
```

Ce minimum est une règle produit.

Il ne signifie pas :

- trois femmes différentes ;
- trois photos explicitement sexuelles ;
- trois récompenses de route ;
- trois images automatiquement obtenues sans contexte.

Une journée peut contenir :

- trois images d’un même set ;
- une image de contexte et deux images d’évolution ;
- un set social ;
- un set privé ;
- une image de conséquence ;
- des alternatives différentes selon la branche.

## 6.2 Garantie par branche

Chaque branche valide doit pouvoir atteindre le minimum visuel.

Une route refusée ou manquée ne doit pas produire une journée vide.

Le remplacement peut être :

- version publique au lieu de privée ;
- image d’autonomie ;
- conséquence ;
- photo du résultat sans accès au processus ;
- set social ;
- image d’un autre pilier de la journée.

Le joueur qui respecte une limite ne doit pas être puni par une absence complète de contenu visuel.

Il peut en revanche perdre :

- une version privée ;
- un niveau d’intensité ;
- une audience ;
- une image supplémentaire ;
- une future progression.

## 6.3 Images individuellement accessibles

Chaque image comptée doit être :

- distincte ;
- consultable individuellement ;
- liée à un identifiant unique ;
- visuellement différente ;
- non dupliquée sous plusieurs identifiants.

Une planche de trois miniatures ne compte pas comme trois images si chaque image ne peut pas être ouverte séparément.

## 6.4 Ce qui ne compte pas

Ne comptent pas dans le minimum :

- pièce vide ;
- verre ;
- nourriture ;
- sac ;
- vêtement sans personne ;
- interface ;
- paysage ;
- document ;
- capture de texte ;
- photographie du décor seule ;
- même image recadrée artificiellement sans nouveau sens ;
- asset dupliqué ;
- placeholder.

Ces éléments peuvent enrichir une scène.

Ils ne remplacent pas une récompense visuelle centrée sur le personnage.

## 6.5 Journée forte

Une journée dominée par une progression visuelle peut contenir :

```text
4 à 6 images
```

Au-delà, chaque image supplémentaire doit justifier :

- une nouvelle intention ;
- une nouvelle audience ;
- une action ;
- une comparaison ;
- une conséquence ;
- une étape réelle de transformation.

## 6.6 Progression dominante

Une journée ou un cycle narratif ne doit généralement avoir qu’une seule progression visuelle dominante.

Il peut également contenir :

- un set social garanti ;
- une trace secondaire ;
- un risque d’audience ;
- un teaser d’une autre route.

Cette règle empêche toutes les routes de devenir simultanément plus explicites.

# 7. Construction d’un set visuel

Un set doit raconter un mouvement.

## 7.1 Set de transformation

```text
avant
→ processus
→ version choisie
→ après
```

Particulièrement adapté à Raphaëlle.

## 7.2 Set d’audience

```text
version publique
→ version privée
→ version secrète ou compromise
```

Particulièrement adapté à Pauline.

## 7.3 Set de regard

```text
version ordinaire
→ regard remarqué
→ effet consciemment choisi
```

Particulièrement adapté à Mathilde.

## 7.4 Set de représentation choisie

```text
image qu’elle tolère ou conserve
→ image qu’elle construit pour elle-même
→ image qu’elle sélectionne pour Player
→ règle sur ce qu’il peut en faire
```

Particulièrement adapté à Sandra.

## 7.5 Set de reconquête

```text
préparation
→ visibilité sociale
→ retour privé
```

Particulièrement adapté à Marie.

## 7.6 Set de conséquence

```text
image initiale
→ nouvelle audience ou découverte
→ image supprimée, conservée ou devenue preuve
```

Adapté aux convergences et fins de route.

## 7.7 Set de scène vécue

```text
approche
→ acte
→ après-coup
```

Utilisé pour les images non diégétiques d’une scène intime ou sexuelle.

# 8. Différenciation des cadres

Les images d’un set doivent varier par au moins deux éléments :

- distance de caméra ;
- orientation du corps ;
- expression ;
- action ;
- relation au regard ;
- environnement ;
- présence d’un tiers ;
- état du vêtement ou du costume ;
- audience ;
- temporalité ;
- fonction.

Ne pas produire :

```text
même pose
+ zoom
+ zoom supplémentaire
```

Un triptyque doit modifier l’interprétation, pas seulement la quantité de pixels occupés par le corps.

# 9. Logique de caméra

Toute photo diégétique doit répondre à :

- Qui tient l’appareil ?
- Pourquoi la photo est-elle prise ?
- Qui peut voir la scène ?
- L’angle est-il physiquement possible ?
- Le miroir est-il cohérent ?
- Un retardateur ou déclencheur est-il établi ?
- La personne sait-elle qu’elle est photographiée ?
- La photo est-elle destinée à quelqu’un ?
- Le photographe conserve-t-il une copie ?

Origines possibles :

- selfie ;
- miroir ;
- retardateur ;
- déclencheur à distance ;
- photographie par un proche nommé ;
- photographie événementielle ;
- capture d’un contenu public ;
- image de groupe ;
- recadrage d’une photo existante ;
- capture d’écran.

Une image explicite ne doit pas être présentée comme une photo diégétique si aucune caméra crédible n’existe.

Dans ce cas, utiliser une **image de scène**.

# 10. Galerie : fonction narrative

La galerie n’est pas un inventaire de corps.

Elle matérialise :

- ce que Player a réellement vu ;
- ce qu’il est autorisé à conserver ;
- ce qui était public ;
- ce qui lui était destiné ;
- ce qui a été partagé ;
- ce qui a été supprimé ;
- ce dont il ne reste qu’une mémoire ;
- la transformation des relations.

Elle ne doit pas afficher :

- score d’affection ;
- nom technique de route ;
- niveau V0–V5 ;
- état caché du personnage ;
- future scène pornographique précise ;
- consentement supposé ;
- silhouette révélant un contenu encore verrouillé.

- un seul contenu ;
- un seul identifiant ;

## 11.3 Ordre

Par défaut :

```text

```

Filtres possibles :

- personnage ;
- public ;
- privée ;
- partagée ;
- souvenir ;
- supprimée ou indisponible ;
- événement.

## 11.4 Libellés joueur

Libellés possibles :

- Publique ;
- Groupe ;
- Pour toi ;
- Partagée ;
- Secrète ;
- Souvenir ;
- Supprimée ;
- Accès perdu.

Les libellés ne doivent refléter que ce que Player sait.

## 11.5 Cases futures

Les emplacements verrouillés peuvent signaler qu’un album possède encore une progression.

Ils ne doivent pas :

- montrer la pose future ;
- révéler la tenue ;
- annoncer la nudité ;
- afficher un intitulé pornographique ;
- transformer la route en checklist.

Utiliser des emplacements discrets et non descriptifs.

# 12. Conservation et mémoire

## 12.1 Image conservable

L’image entre entièrement dans la galerie.

Elle peut être revue selon les règles du cadre.

## 12.2 Image temporaire

Elle peut être vue pendant la scène mais ne reste pas consultable intégralement.

La galerie peut conserver :

- un titre ;
- une date ;
- un souvenir flouté ;
- une vignette abstraite ;
- une mention d’accès perdu.

Elle ne conserve pas l’image complète contre la règle posée.

## 12.3 Image supprimée par Player

La suppression retire l’accès visible.

Elle n’efface pas :

- le souvenir ;
- la conséquence ;
- la connaissance ;
- la réaction du personnage ;
- une éventuelle copie déjà établie.

## 12.4 Image retirée par son auteur

L’image peut devenir :

- inaccessible ;
- remplacée ;
- visible uniquement comme trace ;
- source de manque ou de dette.

## 12.5 Image de scène

Une image de scène peut rester comme souvenir visuel, car elle n’est pas un fichier volé aux personnages.

Elle doit être clairement distinguée d’une photo diégétique dans la logique interne, même si l’interface reste immersive.

# 13. Audience et diffusion

## 13.1 Publique

La photographie est destinée à :

- publication ;
- archive d’événement ;
- groupe ;
- cercle social large.

Public ne signifie pas :

- libre de recadrage sexuel ;
- libre de transfert dans un pacte privé ;
- autorisation d’en faire un contenu pornographique.

## 13.2 Sociale nommée

Audience définie :

- groupe d’amis ;
- personnes présentes ;
- participants d’un événement ;
- famille.

## 13.3 Privée

Image destinée à une seule personne ou un cercle très restreint.

## 13.4 Pour Player

La personne a pensé spécifiquement à son regard.

Cette catégorie représente souvent une progression plus forte que le degré d’exposition.

## 13.5 Partagée

Les destinataires sont nommés et acceptés.

Exemple :

- Player et Nico ;
- Player et Marie ;
- groupe adulte explicitement défini.

## 13.6 Secrète

Les participants savent que l’image est cachée à une personne concernée.

Le secret peut être choisi par les personnes présentes.

Il reste une trahison envers la personne exclue lorsque son accord serait nécessaire.

## 13.7 Audience compromise

L’image a été :

- vue sur le mauvais écran ;
- envoyée au mauvais contact ;
- recadrée ;
- transférée ;
- montrée ;
- découverte.

La conséquence doit être traitée dans la narration.

# 14. Règles de sauvegarde

Valeurs internes recommandées :

```text
AUTORISÉE
INTERDITE
CONDITIONNELLE
NON_DÉCIDÉE
TEMPORAIRE
```

La scène doit préciser :

- sauvegarde automatique ou non ;
- demande nécessaire ;
- autorisation pour une seule image ou tout le set ;
- durée ;
- suppression future promise ;
- conséquence en cas de violation.

Le déverrouillage d’une image dans le jeu ne doit pas signifier automatiquement que Player l’a sauvegardée diégétiquement.

Le jeu doit distinguer :

```text
Player a vu
Player peut revoir comme souvenir
Player possède le fichier
Player peut transférer le fichier
```

# 16. Statut des assets

Valeurs recommandées :

## Prototype

- valide le cadrage ;
- valide le flux ;
- peut contenir des défauts ;
- ne constitue pas une qualité finale.

## Revue

- identité et composition suffisamment stables ;
- continuité à contrôler ;
- encore modifiable.

## Final

- validé visuellement ;
- cohérent avec le canon ;
- sans défaut bloquant ;
- utilisable pour la version publique.

Une image de remplacement, un rectangle ou un placeholder ne peut pas valider une journée visual-first.

# 35. Checklist de validation d’un set

- [ ] le set possède une séquence source ;
- [ ] le personnage est central dans les images comptées ;
- [ ] chaque image est individuellement accessible ;
- [ ] chaque image possède une fonction différente ;
- [ ] l’origine de chaque photo est crédible ;
- [ ] photo diégétique et image de scène sont distinguées ;
- [ ] l’audience est connue ;
- [ ] la sauvegarde est connue ;
- [ ] le transfert est connu ;
- [ ] le niveau d’intensité est cohérent avec l’acte ;
- [ ] le set utilise les objets et lieux du canon ;
- [ ] la progression n’est pas seulement une réduction des vêtements ;
- [ ] le personnage agit et choisit ;
- [ ] le refus possède une mutation ;
- [ ] la route secondaire n’écrase pas la progression dominante ;

- [ ] les images sont visuellement distinctes ;
- [ ] les contrôles de continuité sont satisfaits ;

- [ ] l’après-coup est prévu lorsqu’il est nécessaire.

# 36. Anti-patterns

Rejeter une progression visuelle reposant sur :

- trois variantes du même selfie ;
- exposition corporelle comme seul indicateur de progression ;
- décor présenté comme récompense principale ;
- photo sans origine ;
- caméra impossible ;
- personnage photographié à son insu comme récompense positive ;
- image publique considérée comme librement transférable ;
- conservation automatique de toute image ;
- galerie connaissant davantage que Player ;
- même miroir pour toutes les femmes ;
- même lingerie et même pose pour toutes les routes ;
- Raphaëlle réduite au costume final ;
- Sandra définie par le flou, les objets ou la timidité visuelle ;
- Sandra inondant Player de photos ;
- Sandra gérant des audiences comme Pauline ;
- Sandra utilisant la provocation comme Mathilde ;
- Sandra construisant un rôle comme Raphaëlle ;
- Mathilde supposée séduire parce que son vêtement est ajusté ;
- Pauline sans version publique réelle ;
- Marie uniquement utilisée pour provoquer de la jalousie ;
- Nico créant des doublons pour augmenter le nombre d’images ;
- scène sexuelle photographiée sans appareil ou photographe crédible ;
- image pornographique sans route ni après-coup ;
- refus entraînant toujours une journée visuellement vide ;
- placeholders utilisés pour valider la qualité produit ;
- future nudité révélée par des cases verrouillées ;
- intensité V0–V5 affichée comme statistique au joueur.

# 37. Relation avec le runtime

Ce document définit la direction narrative et visuelle.

Il n’impose pas :

- les noms définitifs des champs ;
- un nouveau système de stockage ;
- une seconde liste d’images déverrouillées ;
- une refonte de la galerie ;
- un système de score visuel ;
- un chargement global différent.

Le runtime futur doit :

- utiliser le système de déverrouillage existant autant que possible ;
- conserver un identifiant unique par contenu ;
- rendre les images réellement visibles ;
- préserver leur ratio ;
- distinguer photo et souvenir lorsque nécessaire ;
- ne pas compter les doublons ;
- permettre les alternatives de branche ;
- respecter les règles de conservation ;
- ne pas exposer les métadonnées internes ;
- échouer proprement en cas d’asset manquant ;
- ne pas considérer un fallback comme une validation visuelle finale.

# 38. Résumé canonique

Les photos sont la récompense visuelle principale de Réseau Intime.

Chaque journée jouable doit offrir au moins trois contenus visuels distincts centrés sur les femmes principales.

Ces images ne sont pas une simple échelle de nudité.

Elles rendent visibles :

- l’intention ;
- l’audience ;
- la confiance ;
- le secret ;
- le risque ;
- la transformation ;
- la conséquence.

Marie progresse de la vie partagée vers la visibilité sociale, la reconquête privée et une sexualité de couple transformée.

Sandra progresse d’une représentation qu’elle a tolérée ou conservée vers une image qu’elle contrôle, sélectionne, destine, puis accepte éventuellement de co-construire avec une personne de confiance.

Mathilde progresse de la sensualité ordinaire vers l’effet consciemment destiné à Player dans le foyer de Marie.

Pauline progresse de la version publique vers le cadrage privé, la preuve et le compartiment.

Raphaëlle progresse de la version professionnelle au processus, à la transformation choisie, au rôle et à ce qui reste après.

Nico ne possède pas les images des femmes. Il modifie leur audience, leur interprétation et leur niveau de risque.

La galerie conserve ce que Player a réellement vu et ce qu’il est autorisé à garder.

Une scène sexuelle peut produire un souvenir visuel sans inventer une photographie diégétique.

Une image importante doit toujours répondre à cette question :

> **Qu’est-ce que cette personne a choisi de montrer, qu’est-ce que Player en fait, et pourquoi leur relation ne peut-elle plus regarder cette image comme avant ?**
