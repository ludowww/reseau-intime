# Réseau Intime — Bible Narrative

> **Statut : Canon — source de vérité narrative principale**
>
> La Bible définit la promesse du jeu, l’expérience joueur, les fantasmes, les transformations, les actes et la bibliothèque de séquences de la première saison.
>
> Les canons complets des personnages restent l’autorité première sur leur identité, leur voix, leur vie concrète, leurs désirs et leurs limites.
>
> Le runtime décrit ce qui est effectivement implémenté et jouable.

---

# 1. Étoile polaire

> **« J’ai eu l’impression de faire lentement basculer plusieurs relations crédibles. »**

Toute décision narrative doit contribuer à cette expérience.

# 2. Hiérarchie officielle

```text
North Star
-> Expérience joueur
-> Fantasmes centraux
-> Canon complet des personnages
-> Routes macro
-> Évolution érotique des routes
-> Actes
-> Séquences
-> Scènes modulaires
-> Dialogues et photos
-> Découpage en journées
-> Runtime
```

Le canon complet des personnages est une autorité transversale.

Il doit être relu :

- avant de définir une route ;
- avant d’ajouter une séquence ;
- avant d’écrire une scène ;
- avant de rédiger un dialogue ;
- avant de concevoir une image ou une progression adulte.

Les journées restent une couche de diffusion, de respiration et de rythme. Elles ne sont jamais la couche principale de conception.

# 4. Différence entre les couches

## North Star

Définit ce que Réseau Intime est et ce qu’il refuse de devenir.

## Expérience joueur

Définit ce que le joueur doit ressentir et attribuer à ses propres actes :

- responsabilité ;
- causalité ;
- simultanéité des relations ;
- risque ;
- limites ;
- conséquences ;
- autonomie des personnages.

## Fantasme

Promesse émotionnelle transversale pouvant être incarnée différemment par plusieurs personnages.

## Canon complet du personnage

Définit :

- identité ;
- passé ;
- proximité initiale ;
- métier ;
- lieux ;
- objets ;
- proches ;
- voix ;
- rapport au corps et aux images ;
- désir ;
- blessure ;
- contradiction ;
- consentement ;
- conséquences propres.

## Route macro

Transformation durable d’une relation.

## Évolution érotique

Décrit comment le désir apparaît, devient intention, trouve un cadre, s’incarne et produit un nouvel état.

## Acte

Transformation dramatique globale de la saison.

## Séquence

Unité concrète accomplissant une ou plusieurs fonctions et transformant un état.

## Scène modulaire

Incarnation précise d’une séquence dans un état de partie donné.

## Dialogues et photos

Incarnent la personne et matérialisent l’audience, la confiance, la limite, le désir ou le risque.

## Journée

Distribue les séquences, silences, respirations et photos.

## Runtime

Décrit ce qui est techniquement implémenté et jouable.

# 5. Règle de spécificité

La grammaire décrit des fonctions réutilisables.

> **Les fonctions peuvent être réutilisées. Les dialogues, réactions, objets, lieux, risques et conséquences doivent rester propres au personnage.**

Une séquence n’est pas spécifique parce que le nom a été remplacé.

Avant de valider une séquence, elle doit utiliser au minimum deux ancrages concrets du canon complet :

- métier ;
- lieu ;
- objet ;
- proche ;
- pratique ;
- blessure ;
- contradiction ;
- moteur érotique.

# 8. Hiérarchie d’autorité

```text
Bible Narrative
= promesse, expérience joueur, fantasmes, structure globale,
  routes, évolution érotique, actes et séquences

Canon complet des personnages
= identité, passé, voix, vie concrète, désir, limites,
  moteur adulte et conséquences individuelles

Contrats narratifs
= choix, modularité, temporalité, communication et adaptation

Runtime
= état jouable, données actives et validations

Archive / À réécrire
= historique ou matériau sans autorité en cas de contradiction
```

En cas de conflit :

- la Bible prévaut sur la direction de la saison ;
- le canon complet prévaut sur l’identité et la singularité du personnage ;
- le NSFW canon prévaut sur les règles adultes générales ;
- le runtime prévaut uniquement sur la description de ce qui est implémenté ;
- un ancien plan par journée ne peut pas invalider une décision de la Bible.

# 9. Moteur considéré comme acquis

Sauf nécessité narrative exceptionnelle démontrée, nous ne revenons pas sur :

- le téléphone ;
- les conversations persistantes ;
- la galerie ;
- les notifications ;
- les photos ;
- les ledgers ;
- les obligations ;
- les scènes modulaires ;
- les tests ;
- l’architecture générale.

La narration est construite au-dessus de ce moteur.

---

# 10. Gouvernance documentaire

Les documents sont classés en quatre catégories :

- **Canon** — référence officielle ;
- **Runtime** — technique et état jouable ;
- **Archive** — historique ;
- **À réécrire** — intention utile devenue incompatible.

Aucun nouveau document ne doit créer une seconde source de vérité sur un sujet déjà couvert.

Les anciens plans de journées peuvent fournir des détails compatibles. Ils ne constituent plus l’architecture principale.
