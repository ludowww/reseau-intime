# Réseau Intime — Bible Narrative

> **Statut : Canon — source de vérité narrative principale**
>
> La Bible définit la promesse du jeu, l’expérience joueur, les fantasmes, les routes, leur évolution érotique, les transformations et l’architecture de la première saison. Le runtime existant est considéré comme acquis et ne doit être remis en question qu’en cas de nécessité narrative exceptionnelle.

---

# 1. Étoile polaire

> **« J’ai eu l’impression de faire lentement basculer plusieurs relations crédibles. »**

Toute décision narrative doit contribuer à cette expérience.

# 4. Différence entre les couches

## North Star

La North Star définit ce que Réseau Intime est et ce qu’il refuse de devenir.

Elle contient la promesse fondamentale et les principes non négociables.

## Expérience joueur

L’expérience joueur définit ce que le joueur doit ressentir et comprendre :

- responsabilité ;
- causalité ;
- simultanéité des relations ;
- risque ;
- limites ;
- conséquences ;
- autonomie des personnages.

Elle sert de test d’acceptation.

## Fantasme

Un fantasme est une promesse émotionnelle transversale.

Il peut être incarné par plusieurs personnages de manière différente.

Exemples : reconquête, secret partagé, jalousie, double vie, interdit domestique.

Un fantasme n’est ni un personnage, ni une route, ni une scène.

## Route macro

Une route macro est la transformation durable d’une relation.

Elle définit :

- une situation de départ ;
- plusieurs états possibles ;
- des résistances ;
- des bascules ;
- des points de retour ou de non-retour ;
- des conséquences sur le réseau ;
- une progression visuelle.

Les routes ne sont jamais présentées comme un menu au joueur.

## Évolution érotique

L’évolution érotique décrit comment le désir apparaît, devient perceptible, trouve un cadre, s’incarne et transforme de nouveau la relation.

Elle distingue :

- évolution relationnelle ;
- degré d’explicitation visuelle ;
- disponibilité ;
- consentement ;
- initiative ;
- après-coup ;
- voies négociées et voies sombres.

Le contenu explicite ne constitue jamais une route à lui seul.

## Acte

Un acte est une transformation dramatique globale de la saison.

Il modifie la pression exercée sur plusieurs relations.

Un acte n’est pas une période calendaire et ne correspond pas automatiquement à un nombre fixe de journées.

## Séquence

Une séquence est une unité dramatique accomplissant une ou plusieurs fonctions.

Elle peut être composée de plusieurs scènes, échanges, photos ou actions hors téléphone.

Exemples de fonctions : CAFÉ, SOIRÉE, RETOUR À LA MAISON, CONFIDENCE.

Les séquences sont conçues avant les journées.

## Scène modulaire

Une scène est l’incarnation concrète d’une séquence dans un état de partie précis.

Elle adapte :

- l’initiateur ;
- les personnages présents ;
- la proximité ;
- les secrets actifs ;
- les obligations ;
- les photos disponibles ;
- le résultat ;
- les conséquences.

Les scènes servent les routes déjà définies.

**Les scènes ne créent pas les routes.**

## Dialogues et photos

Les dialogues incarnent la personne.

Les photos matérialisent une transformation, une audience, une confiance, une limite ou un risque.

Ils sont écrits après la fonction et la transformation attendues.

## Journée

Une journée répartit les séquences, les silences, les respirations et les photos.

Elle sert le rythme et la lisibilité.

Elle ne détermine pas la structure profonde de l’histoire.

## Runtime

Le runtime décrit ce qui est techniquement implémenté et jouable.

Il ne définit pas seul ce que le jeu doit devenir.

# 5. Règle de réutilisation

La grammaire décrit des fonctions réutilisables.

> **Les fonctions de scène peuvent être réutilisées. Les dialogues, les réactions, les risques et les conséquences doivent rester propres au personnage.**

Il est interdit de prendre une scène, de remplacer le nom de Sandra par Mathilde ou Pauline et de considérer la nouvelle version comme spécifique.

La singularité doit venir de :

- la proximité initiale ;
- l’histoire ;
- la voix ;
- les responsabilités ;
- les limites ;
- le rapport au corps et aux images ;
- la manière de prendre l’initiative ;
- le type de risque accepté ;
- les conséquences redoutées.

---

# 6. Architecture hybride validée

```text
Trame principale fixe
-> Routes macro
-> Évolution érotique des routes
-> Actes
-> Séquences
-> Scènes modulaires
-> Dialogues et photos
-> Découpage en journées
-> Runtime
```

La trame fournit les tensions communes.

Les routes définissent les transformations possibles.

L’évolution érotique définit comment le désir devient crédible, visible et incarné.

Les actes organisent la montée dramatique.

Les séquences accomplissent des fonctions.

Les scènes adaptent les séquences à l’état réel de la partie.

Les dialogues et les photos incarnent et matérialisent les changements.

Les journées distribuent le contenu.

# 7. Décisions conservées pour la première saison

La première version est une saison complète mais extensible.

Elle doit fournir une résolution provisoire de la crise principale et transformer toutes les relations majeures.

Marie, Mathilde et Sandra sont les trois premières relations capables d’atteindre une vraie bascule relationnelle et sexuelle.

Pauline porte une promesse forte d’extension et doit obtenir une ligne autonome.

Raphaëlle et Nico sont des graines structurantes dont les conséquences existent déjà sans exiger une route complète dès la première saison.

Nico est hétérosexuel. Il ne possède aucune route romantique ou sexuelle avec Player.

Jeff et Bastien restent des personnages secondaires structurants.

Les photos restent des traces d’évolution et non des récompenses décoratives.

# 8. Hiérarchie d’autorité

```text
Bible Narrative
= promesse, expérience joueur, fantasmes, structure globale,
  routes, évolution érotique, transformations, actes et progression visuelle

Canon complet des personnages
= identité, passé, proximité initiale, voix, contradictions,
  limites, désirs spécifiques et continuité individuelle

Contrats narratifs
= règles de choix, modularité, temporalité, communication et scène

Runtime
= état jouable, contraintes techniques, données actives et validation

Archive / À réécrire
= historique ou matériau sans autorité en cas de contradiction
```

En cas de conflit :

- la Bible prévaut sur la direction du jeu ;
- le canon complet prévaut sur l’identité et la voix, sauf amendement explicite ;
- le runtime prévaut uniquement sur la description de ce qui est implémenté ;
- un ancien plan par journée ne peut pas invalider une décision de la Bible.

---

# 9. Moteur considéré comme acquis

Sauf nécessité narrative exceptionnelle démontrée, nous ne revenons pas sur :

- le téléphone ;
- les conversations persistantes ;
- la galerie ;
- les notifications ;
- les photos ;
- les ledgers ;
- les obligations ;
- le système modulaire ;
- les tests ;
- l’architecture générale.

La narration est construite au-dessus de ce moteur.

---

# 10. Gouvernance documentaire

Les documents doivent être classés en quatre catégories :

- **Canon** — référence officielle ;
- **Runtime** — technique et état jouable ;
- **Archive** — ancien document conservé pour historique ;
- **À réécrire** — intention utile devenue incompatible avec la Bible.

Aucun nouveau document ne doit créer une seconde source de vérité sur un sujet déjà couvert.

Les anciens plans de journées peuvent être utilisés pour retrouver une scène ou un détail compatible. Ils ne constituent plus l’architecture principale.
