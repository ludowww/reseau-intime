# Réseau Intime — 10 Carte des conséquences, dettes, secrets et obligations

## Statut du document

**Catégorie : Canon**

**Niveau : Architecture de la mémoire narrative, des engagements et des retours de conséquence**

Ce document définit comment Réseau Intime mémorise narrativement :

- ce qui s’est réellement passé ;
- ce que chaque personnage sait, soupçonne ou comprend mal ;
- ce qui a été promis ;
- ce qui reste dû ;
- ce qui a été caché ;
- ce qui constitue une dette ;
- ce qui peut devenir une preuve ;
- quand une conséquence doit revenir ;
- comment une trace, une obligation ou un secret cesse d’être actif.

Il complète :

- `05_ROUTES_MACRO_SAISON_1.md` ;
- `06_EVOLUTION_EROTIQUE_DES_ROUTES.md` ;
- `07_BIBLIOTHEQUE_DE_SEQUENCES_SAISON_1.md` ;
- `08_REGLES_DES_SCENES_MODULAIRES.md` ;
- `09_PROGRESSION_VISUELLE_ET_PHOTOGRAPHIQUE.md` ;
- les canons complets des personnages ;

`docs/narrative/PROOF_AND_SECRET_MAP.md`

Ce document ne contient :

# 1. Principe directeur

> **Une conséquence est la forme prise plus tard par un acte qui comptait déjà au moment où il a été accompli.**

Le jeu ne doit pas donner l’impression que les conséquences apparaissent arbitrairement pour punir le joueur.

Elles doivent découler de faits lisibles :

- une promesse ;
- une absence ;
- une limite ;
- une image ;
- une audience ;
- un mensonge ;
- un alibi ;
- une intimité ;
- une attente ;
- une personne exclue ;
- une responsabilité professionnelle, familiale ou conjugale.

Le joueur doit pouvoir comprendre après coup :

```text
j’ai fait cela
→ cette personne l’a compris de cette manière
→ quelque chose est resté dû
→ cette dette a changé la suite
```

Le jeu ne doit pas fonctionner comme :

```text
choix risqué
→ malus invisible
→ punition plusieurs heures plus tard
```

# 2. Les quatre couches à ne pas confondre

## 2.1 Vérité factuelle

Ce qui s’est réellement produit.

Exemples :

- Player est arrivé en retard ;
- Sandra a envoyé une image ;
- Mathilde a choisi une tenue pour produire un effet ;
- Pauline a envoyé deux versions ;
- Nico a vu une photographie ;
- Raphaëlle a nommé un cadre ;
- Marie n’était pas informée ;
- une scène sexuelle a eu lieu ;
- un fichier a été supprimé.

La vérité factuelle existe même si aucun personnage ne la connaît entièrement.

## 2.2 Connaissance

Ce que chaque personnage croit savoir.

La connaissance peut être :

- exacte ;
- partielle ;
- soupçonnée ;
- mal interprétée ;
- niée ;
- confirmée par une preuve.

## 2.3 Engagement

Ce qu’une personne a accepté, promis ou laissé raisonnablement attendre.

Exemples :

- rejoindre Marie ;
- répondre à Sandra ;
- supprimer une image ;
- ne pas transférer un fichier ;
- rendre un objet ;
- revenir après une scène ;
- informer une personne avant une expérience ;
- couvrir Player auprès d’un tiers ;
- tenir un rendez-vous.

## 2.4 Conséquence

Ce que l’histoire doit montrer parce que la vérité, la connaissance ou l’engagement a changé.

Une conséquence peut être :

- positive ;
- négative ;
- ambivalente ;
- protectrice ;
- érotique ;
- sociale ;
- domestique ;
- professionnelle ;
- différée.

Conséquence ne signifie pas automatiquement punition.

# 3. Glossaire canonique

## Fait

Événement réel et stable de l’histoire.

Un fait ne doit pas être réécrit rétroactivement par une nouvelle interprétation.

## Trace

Élément qui rappelle, suggère ou matérialise un fait sans nécessairement le prouver complètement.

Exemples :

- image ;
- message ;
- objet déplacé ;
- horaire ;
- silence ;
- changement d’habitude ;
- vêtement ;
- notification ;
- personne ayant vu quelque chose.

## Preuve

Trace suffisamment forte pour modifier raisonnablement la connaissance ou rendre une version difficile à maintenir.

Une preuve peut être forte sans démontrer toute la vérité.

## Secret

Information volontairement non partagée avec une personne dont la connaissance changerait significativement le cadre relationnel ou moral.

## Vie privée

Information qui n’est pas due à une personne.

Toute information cachée n’est pas un secret problématique.

## Omission

Information pertinente non donnée sans affirmation directement fausse.

## Mensonge

Version activement incompatible avec ce que la personne sait être vrai.

## Promesse

Engagement explicite portant sur une action future.

## Attente légitime

Engagement moins explicite mais suffisamment établi par le contexte, une habitude, un accord ou une initiative pour que son absence produise un effet réel.

## Obligation

Action narrative qui reste due parce qu’un fait, une promesse, une limite ou une scène l’a créée.

## Dette

Asymétrie durable créée lorsqu’une personne a reçu, caché, utilisé ou fait porter quelque chose qui appelle un retour, une réparation, une clarification ou une réciprocité.

Une dette n’est pas une monnaie abstraite.

## Risque

Possibilité crédible qu’une information, une image, un mensonge ou un comportement produise une conséquence.

## Conséquence

Évolution de l’histoire causée par un fait antérieur.

## Après-coup

Conséquence directement liée à une scène intense, adulte, conflictuelle ou transgressive.

## Résolution

Nouvel état suffisamment clair pour que l’obligation, la dette ou le secret n’ait plus besoin de réclamer la même scène.

# 4. Vie privée, secret, omission et mensonge

Ces catégories doivent rester distinctes.

## 4.1 Vie privée légitime

Une personne n’est pas obligée de tout raconter.

Exemples :

- Raphaëlle ne montre pas encore son compte créatif ;
- Sandra garde un selfie pour elle ;
- Mathilde parle à Inès sans informer Player ;
- Marie vit un moment professionnel sans le documenter ;
- Nico ne détaille pas chaque flirt ;
- Pauline garde une pensée personnelle qui n’affecte aucun accord.

La vie privée ne crée pas automatiquement une dette.

## 4.2 Information différée

La personne prévoit raisonnablement d’en parler plus tard ou attend un contexte adapté.

Elle peut devenir une omission si le délai modifie le cadre.

## 4.3 Omission significative

L’information n’est pas donnée alors qu’elle serait pertinente pour :

- un accord ;
- un consentement ;
- une promesse ;
- une décision du couple ;
- un risque d’image ;
- une responsabilité familiale ou professionnelle.

## 4.4 Mensonge actif

Une version fausse est donnée pour empêcher une autre personne d’agir avec les informations réelles.

Exemples :

- faux lieu ;
- faux horaire ;
- faux destinataire d’une image ;
- faux accord de Marie ;
- faux motif professionnel ;
- faux statut d’une relation ;
- fausse suppression d’un fichier.

## 4.5 Secret partagé

Deux personnes savent qu’elles excluent un tiers concerné.

Le secret produit :

- complicité ;
- dette ;
- risque de contradiction ;
- pression de collision ;
- responsabilité commune mais pas nécessairement égale.

## 4.6 Trahison cachée

Le secret concerne un acte qui viole un cadre encore actif.

Le consentement mutuel des participants ne devient jamais celui de la personne exclue.

## 4.7 Cadre privé informé

Les personnes concernées ont accepté que certains détails restent privés.

Ce cadre ne doit pas être confondu avec une trahison simplement bien formulée.

# 5. États de connaissance

Les états suivants sont conceptuels.

Ils ne doivent pas être exposés au joueur comme des étiquettes techniques.

## `INCONNU`

Le personnage ne possède aucun élément pertinent.

## `SOUPÇONNÉ`

Le personnage pense qu’un fait est possible sans pouvoir l’établir.

## `MAL_INTERPRÉTÉ`

Le personnage connaît une trace réelle mais lui attribue une signification incorrecte.

## `CONNU`

Le personnage connaît le fait par observation, information directe ou déduction suffisamment solide.

## `CONFIRMÉ`

Le fait a été explicitement admis ou établi par une preuve difficilement contestable.

## `CONTESTÉ`

Deux personnes ne partagent pas la même version d’un fait malgré la présence d’éléments concrets.

Une scène doit toujours distinguer :

```text
ce qui est vrai
ce que Player sait
ce que le personnage sait
ce que les tiers savent
ce que chacun croit que les autres savent
```

Le dernier niveau ne doit être utilisé que lorsqu’il change réellement une scène.

# 6. Contrat minimal d’une connaissance

Toute connaissance importante doit pouvoir répondre à :

```text
knowledge_id
fact_id
knower
state
source
acquired_when
confidence
what_it_changes
who_knows_the_knower_knows
expiry_or_revision
```

Ce contrat est narratif.

Il n’impose pas un objet runtime complet pour chaque information.

## Source de connaissance

Une connaissance peut venir de :

- observation directe ;
- confession ;
- message ;
- image ;
- témoin ;
- déduction ;
- rumeur ;
- erreur d’audience ;
- contradiction d’horaire ;
- comportement répété.

## Règle

Un personnage ne peut pas commenter précisément :

- une image qu’il n’a pas vue ;
- un message qu’il ne connaît pas ;
- une intention non exprimée ;
- un accord auquel il n’a pas participé ;
- une scène hors téléphone qu’aucune trace ne rend connaissable.

# 7. Typologie canonique des traces et preuves

## 7.1 Trace contextuelle

Établit une présence, un lieu, un rythme ou une relation ordinaire.

Exemples :

- photo de groupe ;
- image de La Verrière ;
- valise de Mathilde ;
- garment bag de Raphaëlle ;
- table réservée à L’Annexe ;
- fin de poste de Sandra.

## 7.2 Trace relationnelle

Montre qu’une personne a pensé à une autre ou qu’une continuité existe.

Exemples :

- image sélectionnée ;
- objet conservé ;
- rendez-vous reprogrammé ;
- message relu ;
- cadre déplacé ;
- siège gardé.

## 7.3 Trace érotisable

Trace encore défendable dont le regard ou le contexte peut devenir sexuel.

Elle n’établit pas automatiquement l’intention de la personne représentée.

## 7.4 Image choisie

Image volontairement créée ou montrée avec une intention et une audience identifiables.

Elle peut être :

- romantique ;
- érotique ;
- provocante ;
- adulte.

## 7.5 Preuve faible

Élément capable de contredire une version sans suffire à établir toute la vérité.

Exemples :

- horaire ;
- notification ;
- message supprimé ;
- changement de ton ;
- photo conservée ;
- absence inexpliquée.

## 7.6 Preuve sociale

Élément visible ou connu de plusieurs personnes.

Il modifie le réseau même si son sens reste ambigu.

## 7.7 Preuve forte

Élément difficilement compatible avec la version officielle.

Exemples :

- message explicite ;
- image destinée ;
- alibi contradictoire ;
- témoignage direct ;
- aveu ;
- objet intime dans un lieu inattendu.

## 7.8 Preuve adulte choisie

Trace explicite créée avec le consentement des participants mais susceptible de devenir dangereuse si son audience change.

## 7.9 Trace compromise

Image, message ou objet sorti de son cadre.

Exemples :

- transfert ;
- recadrage ;
- sauvegarde interdite ;
- mauvais destinataire ;
- écran aperçu ;
- copie non déclarée.

## 7.10 Preuve prohibée

Capture obtenue par caméra cachée, vol, incapacité de consentir ou menace.

Elle ne constitue jamais une récompense positive normale.

---

# 8. Règle de fonction des preuves

Toute preuve doit définir :

```text
qui la possède
qui l’a vue
ce qu’elle prouve
ce qu’elle ne prouve pas
qui pourrait la découvrir
quel mensonge elle contredit
quelle décision elle peut changer
quand elle doit disparaître ou changer de fonction
```

> **Une preuve ne revient pas si elle ne change pas de fonction.**

Une même image peut successivement devenir :

```text
souvenir
→ image destinée
→ secret
→ preuve faible
→ découverte
→ objet de réparation
→ trace retirée ou recontextualisée
```

Elle ne doit pas rester active éternellement uniquement parce qu’elle existe dans une galerie.

# 9. Obligations

Une obligation est créée lorsqu’une scène établit qu’une action future est due.

## 9.1 Promesse d’action

Exemples :

- rejoindre Marie ;
- venir à un événement ;
- rappeler Sandra ;
- aider Raphaëlle ;
- accompagner Mathilde ;
- rencontrer Nico.

## 9.2 Réponse due

Une question ou proposition nécessite une réponse dans une fenêtre crédible.

Le silence peut être une réponse, mais doit produire un état.

## 9.3 Retour relationnel

Après une attention extérieure importante, Player doit parfois :

- revenir au foyer ;
- agir avec Marie ;
- reconnaître qu’il ne répare rien ;
- clarifier une absence.

## 9.4 Obligation d’image

Exemples :

- supprimer ;
- ne pas conserver ;
- ne pas transférer ;
- demander avant de montrer ;
- confirmer la suppression ;
- retirer une publication ;
- corriger une audience.

## 9.5 Obligation de clarification

Un cadre devenu ambigu doit être nommé.

Exemples :

- relation avec Sandra ;
- intention de Mathilde ;
- compartiment Pauline ;
- secret Raphaëlle ;
- rôle de Nico ;
- cadre du couple.

## 9.6 Obligation d’après-coup

Toute scène :

- sexuelle ;
- pornographique ;
- conflictuelle ;
- transgressive ;
- fortement vulnérable ;

peut créer une obligation de :

- message ;
- retour ;
- soin ;
- silence respecté ;
- confirmation du cadre ;
- discussion de la répétition ;
- comportement cohérent le lendemain.

## 9.7 Obligation logistique

Exemples :

- rendre un objet ;
- respecter une heure ;
- libérer une chambre ;
- terminer une réparation ;
- confirmer une réservation ;
- fournir une adresse ;
- organiser un retour.

## 9.8 Obligation professionnelle

Exemples :

- ne pas utiliser un dossier comme prétexte ;
- respecter le retour au travail ;
- protéger une information ;
- ne pas créer de pression hiérarchique ;
- terminer une tâche réelle.

## 9.9 Obligation familiale ou sociale

Exemples :

- préserver la sécurité du foyer ;
- répondre à une cousine hébergée ;
- assumer une absence lors d’un événement ;
- traiter Jeff ou Bastien comme des personnes affectées ;
- ne pas utiliser un proche comme couverture inconsciente.

## 9.10 Obligation d’alibi

Un alibi crée au moins deux obligations :

- cohérence de la version ;
- dette envers la personne qui couvre.

Il peut également créer une obligation envers la personne trompée lorsque la vérité émerge.

---

# 10. Contrat minimal d’une obligation

```text
obligation_id
created_by_fact
owed_by
owed_to
required_action
why_it_is_due
created_when
due_window
hard_deadline
status
allowed_amendments
failure_condition
consequence_if_paid
consequence_if_amended
consequence_if_failed
what_can_cancel_it
```

## Statuts conceptuels

### `PENDING`

Créée mais pas encore exigible.

### `DUE`

La prochaine fenêtre compatible doit permettre son paiement ou sa clarification.

### `OVERDUE`

La fenêtre raisonnable est dépassée.

### `PAID`

L’action promise a été accomplie de manière suffisamment conforme.

### `AMENDED`

L’engagement a été honnêtement renégocié avant ou pendant sa fenêtre.

### `FAILED`

L’action n’a pas été accomplie et aucune modification honnête n’a été acceptée.

### `CANCELLED`

L’obligation n’a plus de raison d’exister à cause d’un changement réel de contexte.

### `TRANSFORMED`

L’obligation initiale devient une autre obligation ou une conséquence.

Ces noms ne doivent pas apparaître au joueur.

---

# 11. Délai, échéance et mutation

Toute obligation doit définir :

- une fenêtre crédible ;
- une éventuelle limite dure ;
- ce qui arrive si elle est différée ;
- ce qui arrive si le contexte disparaît.

## Fenêtre souple

Exemple : répondre à Sandra après un poste difficile.

Un retard peut refroidir la scène sans produire immédiatement une rupture.

## Échéance dure

Exemple : arriver avant que l’événement se termine ou supprimer une image avant qu’elle ne soit publiée.

## Mutation

Une obligation non payée ne réapparaît pas toujours identique.

Exemples :

- Marie profite seule ;
- Sandra retire l’image ;
- Mathilde demande à quelqu’un d’autre ;
- Pauline restaure la surface sans Player ;
- Raphaëlle résout le problème avec Maud ;
- Nico cesse de proposer un alibi ;
- une occasion devient une explication à fournir.

# 12. Dettes

Une dette n’est pas un score que Player rembourse avec une scène agréable ou du sexe.

Elle représente une asymétrie relationnelle précise.

## 12.1 Dette de présence

Une personne a attendu, organisé ou porté seule quelque chose que Player avait accepté de partager.

## 12.2 Dette de confiance

Une personne a montré, confié ou admis quelque chose qui appelle une réponse cohérente.

## 12.3 Dette d’image

Player a reçu un droit limité sur une image.

La dette peut porter sur :

- suppression ;
- confidentialité ;
- absence de transfert ;
- restitution du contrôle ;
- reconnaissance de la personne au-delà du fichier.

## 12.4 Dette d’information

Player sait quelque chose qui affecte une personne ou un accord sans avoir encore décidé comment l’utiliser.

## 12.5 Dette d’alibi

Une personne a menti ou pris un risque pour Player.

Elle peut attendre :

- vérité ;
- réciprocité ;
- protection ;
- fin du pacte ;
- reconnaissance du coût.

## 12.6 Dette domestique

Le foyer, l’hébergement ou une confiance familiale a été utilisé d’une manière qui appelle une règle ou une réparation.

## 12.7 Dette professionnelle

Un accès de travail a été utilisé pour une relation privée ou un incident privé affecte le travail.

## 12.8 Dette sociale

Un groupe, un événement ou une personne a été utilisé comme couverture, témoin ou instrument.

## 12.9 Dette d’après-coup adulte

Une scène intense a créé une vulnérabilité, un nouveau cadre ou une attente qui ne peut pas être laissée sans réponse.

## 12.10 Dette envers soi-même

Un personnage a franchi une limite personnelle et doit décider ce que cela change pour lui.

Cette dette appartient aussi aux personnages non joueurs.

---

# 13. Règles des dettes

1. Une dette possède une origine précise.
2. Une dette possède des personnes concernées.
3. Une dette ne se résout pas par une action sans rapport.
4. Une scène sexuelle ne rembourse pas automatiquement une dette émotionnelle.
5. Une confession ne répare pas automatiquement le dommage révélé.
6. Une suppression ne retire pas une copie ou une connaissance déjà établie.
7. Une dette peut être assumée sans être réparée.
8. Une dette peut transformer une relation sans devenir une punition.
9. Une dette peut être refusée si elle repose sur une demande abusive.
10. Les personnages peuvent agir indépendamment lorsque Player ne paie pas.

---

# 14. Conséquences

## 14.1 Conséquence immédiate

Réaction au sein de la scène ou juste après.

## 14.2 Conséquence différée

Retour dans une fenêtre ultérieure lorsque le contexte rend l’effet visible.

## 14.3 Conséquence comportementale

Changement de :

- ton ;
- disponibilité ;
- initiative ;
- manière de regarder ;
- habitude ;
- choix de tenue ;
- usage du téléphone ;
- retour au foyer.

## 14.4 Conséquence relationnelle

Changement de confiance, accès, attente, limite ou définition.

## 14.5 Conséquence de couple

Effet sur Marie et Player :

- reconnection ;
- dérive ;
- clarification ;
- jalousie ;
- négociation ;
- fracture ;
- rupture ;
- nouvelle règle.

## 14.6 Conséquence visuelle

- image retirée ;
- accès perdu ;
- image recontextualisée ;
- nouvelle version ;
- preuve ;
- galerie modifiée ;
- fichier conservé contre la règle.

## 14.7 Conséquence sociale

Le réseau :

- remarque ;
- commente ;
- se divise ;
- protège ;
- soupçonne ;
- cesse d’inviter ;
- modifie une relation.

## 14.8 Conséquence domestique

Le foyer change :

- porte fermée ;
- chambre quittée ;
- objet rendu ;
- couchage séparé ;
- invitation retirée ;
- habitude modifiée.

## 14.9 Conséquence professionnelle

- distance au travail ;
- confiance réduite ;
- collaboration maintenue avec règles ;
- témoin professionnel ;
- réputation ;
- nécessité de terminer une tâche malgré le privé.

## 14.10 Conséquence familiale

- confiance de Marie envers Mathilde ;
- place de Jeff ;
- place de Bastien ;
- maison des Tilleuls ;
- relations de cousinage ;
- proches devenus témoins.

## 14.11 Conséquence adulte

- après-coup ;
- répétition souhaitée ou refusée ;
- nouvelle pratique ;
- cadre précisé ;
- jalousie ;
- secret ;
- vulnérabilité ;
- distance.

## 14.12 Conséquence de réseau

Une route modifie une autre route ou un contexte partagé.

C’est la forme la plus importante pour éviter six romances parallèles isolées.

# 15. Priorité des conséquences

Lorsqu’une nouvelle scène est possible, l’ordre suivant s’applique.

## A — Sécurité, consentement et retrait

- limite franchie ;
- image diffusée ;
- demande de suppression ;
- personne en insécurité ;
- consentement retiré ;
- cadre adulte devenu invalide.

## B — Après-coup adulte ou conflit majeur

- retour après une scène sexuelle ;
- retour au travail ;
- retour au foyer ;
- soin ;
- nouvelle règle ;
- silence promis.

## C — Obligation à échéance dure

- arrivée ;
- suppression ;
- restitution ;
- événement ;
- réservation ;
- fin de séjour.

## D — Mensonge ou alibi en collision

- contradiction ;
- personne questionnée ;
- preuve apparue ;
- version devenue impossible.

## E — Retour Marie ou conséquence de couple due

Après une attention extérieure majeure, le couple ne doit pas disparaître du récit.

## F — Obligation relationnelle souple

- réponse ;
- clarification ;
- rendez-vous reprogrammé ;
- objet à rendre.

## G — Continuation de route

Une route peut continuer lorsque ses conséquences principales sont traitées.

## H — Nouvelle opportunité

Nouvel accès, nouvelle invitation ou nouveau risque.

## I — Respiration

Vie ordinaire lorsque rien de plus urgent n’est dû.

> **Une nouvelle scène intense ne doit pas servir à éviter une conséquence déjà exigible.**

# 16. Secrets actifs et pression de collision

Un secret actif doit définir :

```text
secret_id
truth
participants
excluded_people
who_knows
who_suspects
supporting_lies
supporting_traces
contradictions
active_obligations
possible_discoveries
what_happens_if_kept
what_happens_if_disclosed
what_happens_if_exposed
```

## Règle de pression

Plusieurs secrets cachés ne peuvent pas s’accumuler indéfiniment sans modifier :

- le temps ;
- les comportements ;
- les horaires ;
- les images ;
- les alibis ;
- le rapport au téléphone ;
- la relation avec Marie ;
- la disponibilité des personnages.

La pression augmente lorsque :

- plusieurs personnes utilisent le même alibi ;
- deux promesses se chevauchent ;
- une image possède plusieurs audiences ;
- un témoin connaît une partie différente ;
- un mensonge exige un second mensonge ;
- le comportement au foyer change ;
- une personne demande une clarification ;
- une preuve doit être supprimée ;
- un partenaire devient directement concerné.

Aucun score de `secret pressure` n’est obligatoire.

La pression doit être lisible par les faits.

# 17. Résolution d’un secret

Un secret peut :

## Être maintenu

Il continue avec une dette et un risque explicites.

## Être confessé

La vérité est donnée par un participant.

La confession ne répare pas automatiquement.

## Être découvert

La vérité ou une partie devient connue sans choix du détenteur.

## Être exposé publiquement

Le réseau reçoit une version.

## Être négocié

Les personnes concernées transforment le cadre.

Une négociation future ne rend pas rétroactivement consentie une trahison passée.

## Devenir sans objet

Exemple : séparation préalable à un acte futur.

Le fait passé et ses conséquences restent vrais.

## Être absorbé

Le secret cesse d’être actif parce que tous les concernés savent et qu’un nouvel état stable existe.

# 18. Alibis

Un alibi doit préciser :

- qui demande ;
- qui couvre ;
- qui est trompé ;
- quel fait est caché ;
- ce que la personne qui couvre sait réellement ;
- quelle version elle doit maintenir ;
- quelle dette est créée ;
- ce qui arrive si elle refuse ;
- ce qui arrive si elle est interrogée.

## Règles

- un alibi ne crée aucun consentement pour l’acte caché ;
- un alibi vague ne peut pas couvrir n’importe quel futur événement ;
- un second alibi augmente la dette ;
- une personne peut accepter de couvrir sans approuver ;
- la personne couverte ne peut pas supposer une loyauté infinie ;
- un alibi peut devenir preuve de complicité.

Nico est particulièrement concerné, mais Pauline, Marie, Mathilde, Raphaëlle ou Sandra peuvent également être impliquées selon le contexte.

# 19. Images, possession et obligations

Player possède le fichier
Player peut transférer le fichier

## Vision autorisée

Aucune possession automatique.

## Souvenir accessible

## Possession autorisée

Player peut conserver selon la règle établie.

Audience supplémentaire explicitement nommée.

## Sauvegarde interdite

Conserver crée une dette et un risque même si personne ne le découvre immédiatement.

## Suppression demandée

L’obligation porte sur :

- l’action ;
- la sincérité ;

- parfois une preuve de suppression.

## Image retirée

L’accès du joueur peut devenir :

- perdu ;
- souvenir ;
- trace grisée ;
- élément de manque ;
- preuve de la limite.

# 28. Refus, retrait et occasion manquée

Refuser une scène ou une route ne crée pas automatiquement une dette.

- dans les temps ;
- sans promesse contradictoire.

- route refroidie ;
- confiance ;
- possibilité future différente ;
- protection du couple ;
- absence de dette.

## Retrait tardif

Peut créer une conséquence si une personne a déjà :

- organisé ;
- attendu ;
- pris un risque ;
- montré une image ;
- modifié sa vie.

Une opportunité expire ou mute.

Elle ne reste pas éternellement offerte.

Les personnages agissent indépendamment.

# 34. Conditions de complétude de la saison

La saison ne peut pas être considérée comme complète si :

- une scène explicite importante n’a aucun après-coup ;
- une image centrale n’a ni statut final ni conséquence ;
- une promesse majeure reste artificiellement suspendue ;
- Jeff ou Bastien est affecté mais traité comme inexistant ;
- le couple Marie / Player n’a pas de définition ;
- un alibi majeur ne produit aucune dette ;
- plusieurs secrets restent empilés sans pression ;
- une violation de consentement ou d’audience est oubliée ;
- un personnage n’a pas repris son agence après une bascule.

Chaque fin doit répondre :

```text
qu’est-ce qui est vrai ?
qui le sait ?
qui croit autre chose ?
qu’est-ce qui reste dû ?
qu’est-ce qui a été perdu ou gagné ?
quelle vie ordinaire existe maintenant ?
```

# 37. Relation avec le runtime

Ce document définit la logique narrative.

Il n’impose pas :

- un objet runtime par fait ;
- une base de données de connaissances complète ;
- une jauge de dettes ;
- un moteur générique de secrets ;
- un nouveau système de sauvegarde ;
- une refonte des ledgers existants.

Le runtime futur doit :

- utiliser les états existants lorsque suffisants ;
- créer uniquement les faits nécessaires aux scènes validées ;
- éviter la duplication entre flags, ledgers, images et historique ;
- conserver des identifiants stables ;
- distinguer état actif et historique ;
- garantir que les obligations dues peuvent revenir ;
- ne pas exposer les noms techniques ;
- tester les collisions importantes ;
- permettre la mutation et l’expiration ;
- préserver les sauvegardes existantes lors de toute évolution.

## Principe de minimalité

```text
une scène validée
→ quelques faits nécessaires
→ une conséquence testable
```

Pas :

```text
un document canonique
→ cent variables génériques ajoutées d’avance
```

# 38. Résumé canonique

Les conséquences ne sont pas des sanctions distribuées après des choix risqués.

Elles sont la mémoire active de ce que les personnages ont fait, promis, caché, compris ou subi.

Une trace indique qu’un fait a laissé quelque chose.

Une preuve modifie ce qu’une personne peut raisonnablement croire.

Un secret exclut quelqu’un dont la connaissance changerait le cadre.

Une obligation décrit ce qui reste à faire.

Une dette décrit ce qui reste déséquilibré.

Une conséquence montre comment cet équilibre change réellement la vie.

Marie transforme les absences et retours en définition du couple.

Sandra transforme le contrôle confié sur sa représentation en confiance ou rupture.

Mathilde transforme l’usage du foyer en règle, secret ou dette familiale.

Pauline transforme les versions et audiences en compartiment, preuve et pouvoir réciproque.

Raphaëlle transforme la précision du cadre en responsabilité avant et après le rôle.

Nico transforme la confidence, l’alibi et le regard partagé en dette d’amitié, rivalité ou garde-fou.

Jeff et Bastien restent des personnes dont la connaissance et les réactions doivent avoir une origine.

Player ne choisit pas seulement ce qu’il désire.

Il choisit également :

- ce qu’il promet ;
- ce qu’il cache ;
- ce qu’il garde ;
- ce qu’il doit rendre ;
- ce qu’il accepte de réparer ;
- et la vérité avec laquelle il peut encore vivre.

> **Une conséquence réussie ne surprend pas parce qu’elle vient de nulle part. Elle frappe parce que le joueur reconnaît exactement le moment où il l’a créée.**
